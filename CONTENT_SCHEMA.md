# Content and Schema Guide

This website is designed so that normal updates happen in `src/data/*.json` and `public/assets/*`, not in React code.

## Daily workflow

```bash
npm install
npm run dev
npm run build
```

Recommended Node version: Node 22 LTS or Node 20 LTS. Avoid experimental Node versions.

Before deployment, run:

```bash
npm run prepare:content
npm run build
```

`prepare:content` scans media folders, extracts optional PDF keywords, and validates content.

### Publication types

Use these strings consistently so categories stay clean:

- `Journal Article`
- `Conference Paper`
- `Workshop Paper`
- `Poster / Extended Abstract`
- `Manuscript Under Review`
- `Manuscript in Preparation`
- `Preprint / Technical Report`

## Keyword system

The keyword dropdown is intentionally conservative now.

### Three keyword fields

1. `tags`
   - Main keywords shown on publication cards/detail pages.
   - Can include many useful descriptive terms.

2. `filterKeywords`
   - Small curated list used for the publication dropdown filter.
   - Recommended: 3–5 terms per paper.
   - Because `useExplicitFilterKeywordsOnly` is enabled, only these terms populate the dropdown.

3. Auto PDF keywords
   - Generated by `npm run extract:keywords` from local PDFs.
   - These appear on publication detail pages by default, but **do not enter the filter dropdown** unless enabled.

### Removing a keyword permanently

Edit: `src/data/keyword-config.json`

To remove a keyword everywhere:

```json
"excludeKeywords": ["unwanted keyword"]
```

To remove only from the publication filter dropdown:

```json
"excludeFilterKeywords": ["unwanted keyword"]
```

To prevent PDF extraction from re-adding a keyword:

```json
"excludeAutoKeywords": ["unwanted keyword"]
```

To hide a keyword only for one publication:

```json
"hiddenKeywords": ["unwanted keyword"]
```

To hide an auto-generated keyword only for one publication:

```json
"hiddenAutoKeywords": ["unwanted keyword"]
```

To hide a dropdown keyword only for one publication:

```json
"hiddenFilterKeywords": ["unwanted keyword"]
```

### Merging/capitalizing keywords

Use `aliases` in `keyword-config.json`:

```json
"aliases": {
  "reeb spaces": "Reeb Space",
  "csp": "continuous scatterplots",
  "ct": "computed tomography"
}
```

The display capitalization is normalized automatically, with common acronyms such as `AI`, `ML`, `CT`, `TVCG`, and `IEEE` preserved.

## Publication PDFs, supplementary material, videos, and certificates

You can use external URLs or local asset paths.

Local paper PDF:

```json
"localPdf": "/assets/publications/pdf/paper-id.pdf"
```

External paper PDF:

```json
"pdf": "https://.../paper.pdf"
```

Supplementary materials can be any number of items:

```json
"supplements": [
  { "id": "supplement-1", "label": "Supplementary PDF", "type": "pdf", "path": "/assets/publications/supplements/file.pdf", "preview": true },
  { "id": "dataset", "label": "Dataset", "type": "link", "url": "https://...", "preview": false }
]
```

Videos can be YouTube/Vimeo/local files:

```json
"videos": [
  { "id": "video-1", "label": "Talk Video", "url": "https://www.youtube.com/watch?v=...", "preview": true }
]
```

Award certificates can be previewed like PDFs:

```json
"awards": [
  { "id": "certificate", "label": "Best Poster Award", "type": "pdf", "path": "/assets/publications/certificates/best-poster.pdf", "preview": true }
]
```

Choose the default preview on the publication detail page:

```json
"defaultPreview": "paper"
```

or

```json
"defaultPreview": "video-1"
```

or

```json
"defaultPreview": "certificate"
```

## Homepage media highlight

Edit: `src/data/home-media.json`

Use a 16:9 image or video for best results.

```json
{
  "enabled": true,
  "defaultItemId": "recent-talk",
  "items": [
    {
      "id": "recent-talk",
      "type": "youtube",
      "title": "Recent talk",
      "url": "https://www.youtube.com/watch?v=...",
      "caption": "Short caption shown below the media.",
      "autoplayMuted": true,
      "link": "#talks",
      "linkText": "View talks"
    }
  ]
}
```

Supported `type`: `image`, `video`, `youtube`.

## News

Edit: `src/data/news.json`

News cards are clickable. Use internal routes when possible.

```json
{
  "date": "2025-09-01",
  "title": "Best Poster Award at VCBM",
  "description": "Short description.",
  "route": "publication:spectral-ct-vcbm-poster-2025",
  "url": "https://..."
}
```

Priority:

1. `route` opens an internal page.
2. `url` opens an external page.
3. If neither exists, the card is static.

## Deployment to GitHub Pages

1. Build locally:

```bash
npm run build
```

2. Push to GitHub.
3. Use the included GitHub Actions workflow if present, or deploy the `dist/` folder.

If your repository is named `username.github.io`, set `base: './'` in `vite.config.js` as currently configured.
